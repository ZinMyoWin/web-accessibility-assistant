# Documentation Index

This folder contains the main project documentation.

Use this structure when adding new documents:

- `architecture/`
  - technical design and system structure
- `guides/`
  - setup guides and repeatable operational instructions
- `implementation/`
  - feature implementation plans and beginner-friendly build notes
- `tracking/`
  - progress tracking, checklists, and implementation history

## Documentation Rule

When a meaningful feature is added or changed:

1. update the architecture document if the design changes
2. update the implementation guide if the feature needs beginner-friendly build notes
3. update the checklist if feature status changes
4. update the implementation log with what was done and how it was verified
